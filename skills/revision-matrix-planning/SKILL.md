# Revision Matrix Planning

Use this skill when review comments, advisor feedback, or manuscript critique
need to become an auditable revision plan.

## Allowed Mutation Mode

Artifact mutation for revision matrices and response plans when authorized.
Do not edit manuscript text directly; route approved text changes to LaTeX
paper authoring.

## Artifact Inputs

- Review package or feedback list
- Manuscript root or draft location references
- Claim register
- Source manifest
- Author decisions or constraints

## Workflow

1. Normalize each feedback item into issue, location, severity, and requested
   change.
2. Assign response stance: accept, partially accept, reject with rationale, or
   needs human decision.
3. Identify affected claims, sources, figures, tables, and manuscript sections.
4. Produce a revision matrix with owner decision points and verification needs.
5. Route approved text edits to LaTeX authoring.

## Artifact Outputs

- Revision matrix
- Response plan
- Change-risk notes
- Human decision list

## Source Support

When a revision changes a factual claim, tie the planned change to source
manifest entries or claim-register support. Flag unsupported new claims before
they reach manuscript editing.

## Human Checkpoints

The human researcher approves response stance, rejected-review rationale, and
any manuscript mutation plan before edits are made.
