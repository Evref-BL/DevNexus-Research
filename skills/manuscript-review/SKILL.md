# Manuscript Review

Use this skill when the user wants critique, advisor-style feedback, reviewer
perspective, or readiness assessment without directly changing the manuscript.

## Allowed Mutation Mode

Read-only. Do not edit manuscript files, bibliography files, claim registers,
or source manifests. Produce review artifacts and route approved changes to a
revision or LaTeX authoring workflow.

## Artifact Inputs

- Manuscript root or draft export
- Research brief
- Claim register
- Source manifest
- Venue, assignment, or reviewer criteria

## Workflow

1. Confirm review lens: clarity, structure, novelty, method, evidence, venue,
   reproducibility, or reviewer risk.
2. Read the manuscript against the requested lens.
3. Separate blocking issues, important improvements, and optional polish.
4. Tie critique to manuscript locations and artifact evidence where possible.
5. Recommend revision-matrix planning for accepted changes.

## Artifact Outputs

- Review package
- Reviewer risk list
- Revision candidates
- Open questions for the author

## Source Support

Do not assert that a claim is unsupported unless the relevant manifest,
bibliography, or supplied source context was checked. Mark uncertainty and name
the artifact needed to resolve it.

## Human Checkpoints

The human researcher decides which critiques are valid, which become revision
work, and which authorial choices should remain unchanged.
